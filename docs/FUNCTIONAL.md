## 2. Getting Started

### Opening the Application
1. Open your web browser (Chrome, Firefox, Safari, or Edge)
2. Navigate to: **https://maheshyadav-hiq.github.io/job-tracker/**
3. The application will load automatically

### First Time Use
When you first open JobTrack Pro, you'll see an empty state with a prompt to add your first job application. Click the **"Add Your First Application"** button to get started.

## 3. Adding a Job Application

### Step-by-Step Guide

1. **Click "Add Job"** - Located in the top-right corner of the header

2. **Fill in the Details** - Here's what each field means:

   | Field | Required | Description |
   |-------|----------|-------------|
   | Job Title | ‚úÖ Yes | The position you're applying for (e.g., "Senior React Developer") |
   | Company Name | ‚úÖ Yes | The company you're applying to (e.g., "Google") |
   | Company URL | No | The company's website (e.g., "https://google.com") |
   | Job Listing URL | No | The direct link to the job posting |
   | Status | No | Current status (defaults to "Wishlist") |
   | Salary Range | No | Expected salary (e.g., "$120k - $150k") |
   | Location | No | Job location (e.g., "San Francisco, CA" or "Remote") |
   | Date Applied | No | When you submitted the application |
   | Interview Date | No | Scheduled interview date |
   | Notes | No | Any additional information |

3. **Click "Save Application"** - Your application will be added to the list

### Tips for Adding Applications
- Always fill in Job Title and Company Name (these are required)
- Add the application as soon as you apply so you don't forget
- Use the Notes field to track important details like recruiter name or next steps

## 4. Managing Your Applications

### Viewing Applications
Your applications appear as cards on the main dashboard. Each card shows:
- Company name and avatar
- Job title
- Date applied
- Current status

### Editing an Application
1. Click on the **edit icon** (pencil) on the application card
2. Modify any fields in the form
3. Click "Save Application" to confirm changes

### Deleting an Application
1. Click on the **trash icon** on the application card
2. A confirmation dialog will appear
3. Click "Delete" to confirm (or "Cancel" to abort)

> ‚ö†Ô∏è **Warning:** Deleting an application cannot be undone. Make sure you want to delete it before confirming.

### Changing Status Quickly
Instead of opening the edit form, you can quickly change status by:
1. Click directly on the **status badge** on any application card
2. The status will cycle to the next stage

**Status Order:** Wishlist ‚Üí Applied ‚Üí Interview ‚Üí Offer ‚Üí Rejected ‚Üí (cycles back)

## 5. Using Filters and Search

### Filtering by Status
Use the filter tabs at the top of the application list:
- **All** - Shows all applications
- **Wishlist** - Companies you're interested in but haven't applied to
- **Applied** - Applications you've submitted
- **Interview** - Applications with scheduled interviews
- **Offer** - Applications where you received an offer
- **Rejected** - Applications that were not successful

### Searching
Use the search box to find specific applications:
1. Type in the search field (job title or company name)
2. Results update instantly as you type
3. Click "Clear Filters" to reset

### Sorting Applications
Change how applications are ordered using the dropdown:
- **Sort by Date** - Most recent first (default)
- **Sort by Company** - Alphabetically by company name
- **Sort by Status** - By application status order

## 6. Understanding the Dashboard

### Stats Cards
The dashboard shows four key metrics:

| Stat | Description |
|------|-------------|
| **Total Applications** | All applications you've added |
| **Active** | Applications in "Applied" or "Interview" status |
| **Interviews** | Applications with scheduled interviews |
| **Offers** | Applications where you received an offer |

### What Counts as "Active"?
"Active" means you're still in the running. This includes:
- Applications you've submitted (Applied)
- Applications with scheduled interviews (Interview)

## 7. Data Storage

### How Your Data is Stored
Your job applications are saved automatically to your browser's local storage. This means:
- ‚úÖ Data persists when you close and reopen the browser
- ‚úÖ Data persists when you refresh the page
- ‚ùå Data is stored only on this specific device/browser

### Important Notes
- **One device at a time** - Your data lives only on the device you're using
- **Browser-specific** - Data won't transfer between Chrome and Firefox
- **No account needed** - Everything stays private on your device

## 8. Troubleshooting

### Common Issues

| Problem | Solution |
|---------|----------|
| Application won't save | Make sure Job Title and Company Name are filled in |
| Data disappeared | Check if you're using a different browser or cleared your cache |
| Search not working | Make sure you're typing the exact job title or company name |
| Can't see the application | Check if a filter is active (try clicking "All") |

### Need Help?
If you encounter any issues:
1. Try refreshing the page
2. Clear your browser cache and try again
3. Contact support if the problem persists
